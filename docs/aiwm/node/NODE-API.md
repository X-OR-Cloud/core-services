# Node Module - Entity & API Reference

> Last updated: 2026-03-10

---

## 1. Entity: Node

### Base fields (inherited from BaseSchema)

| Field | Type | Description |
|-------|------|-------------|
| `_id` | ObjectId | MongoDB document ID |
| `owner` | `{ userId, orgId }` | Creator's user ID and organization ID |
| `createdBy` | ObjectId | User who created the node |
| `updatedBy` | ObjectId | User who last updated the node |
| `isDeleted` | boolean | Soft delete flag (default: false) |
| `metadata` | object | Arbitrary key-value metadata |
| `createdAt` | Date | Auto-generated by Mongoose timestamps |
| `updatedAt` | Date | Auto-generated by Mongoose timestamps |

### Core fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Human-readable node name, e.g. `gpu-node-01` |
| `role` | string[] | No | Node roles: `controller`, `worker`, `proxy`, `storage`. A node can hold multiple roles. |
| `status` | string | Yes | Node lifecycle status (see table below). Default: `pending` |
| `lastHeartbeat` | Date | No | Last heartbeat received from node agent. Default: creation time |
| `lastMetricsAt` | Date | No | Last time node pushed metrics |

### Status values

| Status | Description | Set by |
|--------|-------------|--------|
| `awaiting-approval` | Created by non-org.owner — waiting for owner to approve | System on create |
| `pending` | Approved/created by owner — ready for setup | System on create / approve |
| `installing` | Install script is running (bootstrap in progress) | bootstrap API |
| `online` | Node agent connected via WebSocket | WebSocket gateway on connect |
| `offline` | Node agent disconnected | WebSocket gateway on disconnect |
| `maintenance` | Manually taken offline for maintenance | Future: manual update |

**State machine:**
```
org.owner creates     → pending
non-owner creates     → awaiting-approval
org.owner approves    → pending
bootstrap called      → installing
ws.connect + heartbeat → online
ws.disconnect         → offline
```

### Authentication fields

| Field | Type | Description |
|-------|------|-------------|
| `secretHash` | string | bcrypt hash of node secret. Never returned in API responses. |
| `lastAuthAt` | Date | Last time node authenticated via login |
| `setupTokenHash` | string | sha256 hash of the current setup JWT. Cleared after use. Never returned in API. |
| `setupTokenExpiresAt` | Date | Expiry time of the setup token |
| `apiKey` | string | **(Deprecated)** Legacy UUID used by old node agents. Use `_id` instead. |

### SystemInfo (nested object)

Populated by node agent after first WebSocket `node.register` event.

```
systemInfo:
  os:
    name: string              # e.g. "Ubuntu"
    version: string           # e.g. "22.04 LTS"
    kernel: string            # e.g. "5.15.0-91"
    platform: string          # e.g. "linux"
  architecture:
    cpu: string               # e.g. "x86_64"
    bits: number              # e.g. 64
    endianness: string        # e.g. "LE"
  hardware:
    cpu:
      model: string           # e.g. "Intel Xeon Gold 6348"
      vendor: string          # e.g. "Intel"
      sockets: number
      coresPerSocket: number
      threadsPerCore: number
      totalCores: number
      frequency: number       # MHz
      cacheSize: number       # KB
      details[]:              # Per-socket details
        socketId, model, vendor, frequency, cacheSize, cores
    memory:
      total: number           # bytes
    disk:
      total: number           # bytes
      devices[]:
        name, mountPoint, total, filesystem
    network:
      publicIp: string
      clusterIp: string
      ports: Record<string, number>
      interfaces[]:
        name, type, macAddress, ipAddress, ipv6Address, netmask,
        gateway, dns[], mtu, speed, duplex, state, isPrimary,
        isVirtual, parentInterface, vlanId
      connectivity:
        hasInternet: boolean
        publicIpDetected: string
        lastChecked: Date
        reachableFrom: string[]
    gpu[]:
      deviceId: string
      model: string           # e.g. "NVIDIA A100"
      vendor: string
      memoryTotal: number     # bytes
      capabilities: string[]
  containerRuntime?:
    type: string              # e.g. "docker"
    version: string
    apiVersion: string
    storage: { driver, filesystem }
  virtualization?:
    type: string              # e.g. "kvm"
    role: string              # e.g. "guest"
```

## 2. API Endpoints

### Authentication

| Endpoint | Auth required |
|----------|--------------|
| `POST /nodes` | User JWT (`organization.editor` or `organization.owner`) |
| `GET /nodes` | User JWT |
| `GET /nodes/:id` | User JWT |
| `POST /nodes/:id/approve` | User JWT (`organization.owner` only) |
| `POST /nodes/:id/setup-guide` | User JWT (`organization.owner` or node creator) |
| `POST /nodes/:id/regenerate-credentials` | User JWT (`organization.owner` or node creator) |
| `POST /nodes/auth/bootstrap` | Public (setup token in body) |
| `POST /nodes/auth/login` | Public (node credentials in body) |
| `POST /nodes/auth/refresh` | Public (JWT in body) |

### POST /nodes — Create node

**Auth:** User JWT. Role: `organization.editor` or `organization.owner`.

**Request body:**
```json
{
  "name": "gpu-node-01",
  "role": ["worker", "storage"]
}
```

| Field | Type | Required | Values |
|-------|------|----------|--------|
| `name` | string | Yes | Node display name |
| `role` | string[] | Yes | `controller`, `worker`, `proxy`, `storage` |

**Response 201 — org.owner creates (status: pending):**
```json
{
  "_id": "507f1f77bcf86cd799439011",
  "name": "gpu-node-01",
  "role": ["worker", "storage"],
  "status": "pending",
  "owner": {
    "userId": "691eb9e6517f917943ae1f9d",
    "orgId": "691eb9e6517f917943ae1f9a"
  },
  "createdAt": "2026-03-10T08:00:00.000Z",
  "updatedAt": "2026-03-10T08:00:00.000Z"
}
```

**Response 201 — non-owner creates (status: awaiting-approval):**
```json
{
  "_id": "507f1f77bcf86cd799439011",
  "name": "gpu-node-01",
  "role": ["worker"],
  "status": "awaiting-approval",
  "owner": {
    "userId": "691eb9e6517f917943ae1f9d",
    "orgId": "691eb9e6517f917943ae1f9a"
  },
  "createdAt": "2026-03-10T08:00:00.000Z",
  "updatedAt": "2026-03-10T08:00:00.000Z"
}
```

**Response 400 — Insufficient role:**
```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Insufficient permissions to create node"
}
```

### POST /nodes/:id/approve — Approve node

**Auth:** User JWT. Role: `organization.owner` only.

**Description:** Transitions node from `awaiting-approval` → `pending`. Node can then proceed with setup guide.

**Request body:** _(empty)_

**Response 200 — success:**
```json
{
  "_id": "507f1f77bcf86cd799439011",
  "name": "gpu-node-01",
  "status": "pending",
  "updatedAt": "2026-03-10T08:05:00.000Z"
}
```

**Response 400 — node not awaiting approval:**
```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Node is not awaiting approval (current status: pending)"
}
```

**Response 403 — not org.owner:**
```json
{
  "statusCode": 403,
  "error": "Forbidden",
  "message": "Only organization owner can approve nodes"
}
```

**Response 404:**
```json
{
  "statusCode": 404,
  "error": "Not Found",
  "message": "Node not found"
}
```

### POST /nodes/:id/setup-guide — Get setup guide

**Auth:** User JWT. Must be `organization.owner` or the node creator.

**Description:** Generates a one-time setup token (24h JWT) and returns installation instructions. Node must be in `pending` status.

The setup token is a JWT containing all necessary connection URLs so the install script can self-configure.

**Request body:**
```json
{
  "os": "ubuntu"
}
```

| Field | Type | Required | Values |
|-------|------|----------|--------|
| `os` | string | Yes | `ubuntu` |

**Response 200:**
```json
{
  "os": "ubuntu",
  "installCommand": "curl -fsSL http://releases.x-or.cloud/xor-stack-ai-node-agent-install.sh | bash -s -- --token eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "instructions": [
    "1. Make sure you have curl installed: apt-get install -y curl",
    "2. Run the install command below as root or with sudo",
    "3. The script will automatically configure and start the node agent",
    "4. After installation, the node status will change to 'installing' then 'online'"
  ],
  "setupTokenExpiresAt": "2026-03-11T08:10:00.000Z"
}
```

**Setup token JWT payload (decoded — for reference):**
```json
{
  "nodeId": "507f1f77bcf86cd799439011",
  "bootstrapUrl": "http://localhost:3003/nodes/auth/bootstrap",
  "apiBaseUrl": "http://localhost:3003",
  "wsBaseUrl": "ws://localhost:3003",
  "wsPath": "/ws/node",
  "monaBaseUrl": "http://localhost:3005",
  "iat": 1741593000,
  "exp": 1741679400
}
```

**Response 400 — node not in pending status:**
```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Node must be in pending status to generate setup guide (current: awaiting-approval)"
}
```

**Response 403 — not owner or creator:**
```json
{
  "statusCode": 403,
  "error": "Forbidden",
  "message": "Only organization owner or node creator can generate setup guide"
}
```

**Response 404:**
```json
{
  "statusCode": 404,
  "error": "Not Found",
  "message": "Node not found"
}
```

### POST /nodes/:id/regenerate-credentials — Regenerate node secret

**Auth:** User JWT. Must be `organization.owner` or the node creator.

**Description:** Generates a new secret for the node. Old secret is immediately invalidated. Node must be in `online` or `offline` status. Returns a warning if the node is currently online (will disconnect immediately).

**Request body:** _(empty)_

**Response 200 — node is offline:**
```json
{
  "nodeId": "507f1f77bcf86cd799439011",
  "credentials": {
    "secret": "b8c3d4e5-f6g7-5h8i-9j0k-1l2m3n4o5p6q"
  },
  "warning": null,
  "affectedStatus": "offline"
}
```

**Response 200 — node is online (warning returned):**
```json
{
  "nodeId": "507f1f77bcf86cd799439011",
  "credentials": {
    "secret": "b8c3d4e5-f6g7-5h8i-9j0k-1l2m3n4o5p6q"
  },
  "warning": "WARNING: This node is currently ONLINE. Resetting the secret will immediately disconnect the running node.",
  "affectedStatus": "online"
}
```

> **Important:** The secret is shown only once. Save it immediately — it cannot be retrieved again.

**Response 400 — node not online or offline:**
```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Credentials can only be regenerated for online or offline nodes (current: installing)"
}
```

**Response 403:**
```json
{
  "statusCode": 403,
  "error": "Forbidden",
  "message": "Only organization owner or node creator can regenerate credentials"
}
```

**Response 404:**
```json
{
  "statusCode": 404,
  "error": "Not Found",
  "message": "Node not found"
}
```

### POST /nodes/auth/bootstrap — Bootstrap node

**Auth:** Public (no JWT required — setup token in body).

**Description:** Called by the install script after download. Exchanges the setup token for the node secret (shown ONCE). Node status transitions to `installing`.

The setup token is a one-time-use JWT. It is invalidated immediately after use.

**Request body:**
```json
{
  "setupToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `setupToken` | string | Yes | JWT token obtained from setup guide install command |

**Response 200 — success:**
```json
{
  "nodeId": "507f1f77bcf86cd799439011",
  "secret": "e95ec1e2-a295-4373-972d-0db949df7e2a",
  "warning": "This secret is shown ONLY ONCE. Save it securely — it cannot be retrieved again."
}
```

**Response 401 — invalid or expired token:**
```json
{
  "statusCode": 401,
  "error": "Unauthorized",
  "message": "Invalid or expired setup token"
}
```

**Response 401 — token already used:**
```json
{
  "statusCode": 401,
  "error": "Unauthorized",
  "message": "Setup token has already been used"
}
```

**Response 403 — node not yet approved:**
```json
{
  "statusCode": 403,
  "error": "Forbidden",
  "message": "Node has not been approved yet (status: awaiting-approval)"
}
```

**Response 404:**
```json
{
  "statusCode": 404,
  "error": "Not Found",
  "message": "Node not found"
}
```

### POST /nodes/auth/login — Node login

**Auth:** Public.

**Description:** Authenticates a node agent using its ID + secret. Returns a JWT token for WebSocket connection.

Supports two modes:
- **New nodes (ObjectId):** Pass `id` = MongoDB `_id` (24-char hex)
- **Legacy nodes (UUID):** Pass `id` or `apiKey` = UUID v4 (backward compatibility)

**Request body:**
```json
{
  "id": "507f1f77bcf86cd799439011",
  "secret": "e95ec1e2-a295-4373-972d-0db949df7e2a"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | One of `id`/`apiKey` | ObjectId (new) or UUID (legacy) |
| `apiKey` | string | One of `id`/`apiKey` | **(Deprecated)** Legacy UUID. Use `id` instead. |
| `secret` | string | Yes | Node secret |

**Response 200:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresIn": 3600,
  "tokenType": "Bearer",
  "node": {
    "_id": "6942690ddfc4396c89d3a3ea",
    "name": "gpu-node-01",
    "status": "online",
    "roles": ["worker", "storage"],
    "orgId": "691eb9e6517f917943ae1f9d"
  }
}
```

**Response 401 — invalid credentials:**
```json
{
  "statusCode": 401,
  "error": "Unauthorized",
  "message": "Invalid node credentials"
}
```

### POST /nodes/auth/refresh — Refresh node token

**Auth:** Public (current JWT in body).

**Description:** Exchanges an existing JWT for a new one with extended expiration. Allowed within a 5-minute grace period after expiry.

**Request body:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `token` | string | Yes | Current access token (valid or recently expired) |

**Response 200:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.new...",
  "expiresIn": 3600,
  "tokenType": "Bearer"
}
```

**Response 401 — expired beyond grace period:**
```json
{
  "statusCode": 401,
  "error": "Unauthorized",
  "message": "Token expired and grace period has passed"
}
```

**Response 401 — invalid token:**
```json
{
  "statusCode": 401,
  "error": "Unauthorized",
  "message": "Invalid token"
}
```

## 3. Error Code Reference

| HTTP Status | Scenario | Message pattern |
|-------------|----------|-----------------|
| 400 | Node not in correct status for operation | `Node must be in <X> status ...` |
| 400 | Insufficient permissions to create | `Insufficient permissions to create node` |
| 401 | Invalid node credentials (login) | `Invalid node credentials` |
| 401 | Invalid/expired setup token | `Invalid or expired setup token` |
| 401 | Setup token already used | `Setup token has already been used` |
| 401 | JWT expired beyond grace period | `Token expired and grace period has passed` |
| 403 | Not org.owner (approve) | `Only organization owner can approve nodes` |
| 403 | Not org.owner or creator (setup-guide / regenerate) | `Only organization owner or node creator can ...` |
| 403 | Node not approved (bootstrap) | `Node has not been approved yet (status: awaiting-approval)` |
| 404 | Node not found | `Node with ID <id> not found` or `Node not found` |

## 4. Node Provisioning Flow

```
1. User (org.editor or org.owner) calls POST /nodes
   └─► org.owner → status: pending
   └─► org.editor → status: awaiting-approval

2. [If awaiting-approval] org.owner calls POST /nodes/:id/approve
   └─► status → pending

3. org.owner or creator calls POST /nodes/:id/setup-guide { os: "ubuntu" }
   └─► Server generates setupToken JWT (24h)
   └─► Returns installCommand with embedded token

4. User runs installCommand on target machine
   └─► Install script decodes setupToken to get bootstrapUrl, apiBaseUrl, etc.
   └─► Script calls POST /nodes/auth/bootstrap { setupToken }
       └─► Server returns { nodeId, secret }
       └─► Node status → installing
       └─► Token is invalidated (one-time use)

5. Node agent saves nodeId + secret, then calls POST /nodes/auth/login
   └─► Returns JWT accessToken (1h)

6. Node agent connects to WebSocket ws://.../ws/node (JWT in auth.token)
   └─► Gateway verifies JWT → status → online
   └─► Node sends node.register with systemInfo
   └─► Node starts heartbeat loop (every 30s)
```
